Attack Scenarios
Browser MQTT Injector
DDoS — UDP Flood
Effect: Exhausts target bandwidth via high-volume UDP burst.
Detection: Zero-variance packet size, high IAT frequency, zero ACK/SYN flags.
DoS — SYN Flood
Effect: Exhausts TCP connection queue with half-open connections.
Detection: High SYN count, low ACK ratio (<0.3), many RST responses.
Mirai — IoT Botnet
Effect: Multi-vector flood mimicking Mirai botnet C2 coordination.
Detection: UDP + large packet size (≥500B) + high packet density.
Spoofing — ARP/MITM
Effect: Poisons ARP cache to intercept LAN traffic (Man-in-the-Middle).
Detection: Non-TCP/UDP/ICMP protocol, tiny packet size (<60B), repeated source.
Recon — Port Scan
Effect: Maps open ports and running services on target node.
Detection: High RST count, small average payload size (<150B), sequential port probing.

Training Methodology

Dataset Source:
Canadian Institute for Cybersecurity (CIC-IoT 2023). Contains 33 attacks across 7 categories, executed against 105 real IoT devices.

Feature Engineering:
Since the ESP32 captures traffic promiscuously (encrypted WPA2), standard payload inspection is impossible. The model relies entirely on 15 L2/L3/L4 statistical headers (size, rate, timing, TCP flags).

Hyperparameters:
